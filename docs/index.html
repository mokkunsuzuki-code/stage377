Stage377

Production Dual-Timestamp Finalization and Superseding Final Acceptance Gate

RFC3161とOpenTimestampsが同一のStage360対象を それぞれ独立に検証した場合のみ、 新しい有効な最終承認を成立させます。

Fail-Closed Initial State: timestamp_finalization_pending

Stage377の目的

Stage377は、Stage376で定義された 二重タイムスタンプ最終承認モデルを、 実際の外部検証レシートへ接続する最終化レイヤーです。

RFC3161

RFC3161タイムスタンプ局から取得した応答について、 OpenSSLを使い、対象ハッシュ、署名、 証明書チェーンを検証します。

  • SHA256対象固定
  • TSA署名検証
  • 証明書チェーン検証
  • 検証メタデータのみ公開

OpenTimestamps

同じStage360対象についてOpenTimestamps証明を作成し、 公開ブロックチェーンへの確定状態を確認します。

  • SHA256対象固定
  • 公開アンカー確認
  • 確定時刻確認
  • 未確定時はpending

現在の初期状態

Decision: timestamp_finalization_pending
RFC3161 Verified: False
OpenTimestamps Verified: False
Effective Final Acceptance: False
Maximum Timestamp Assurance: False

外部証明がまだ検証されていないため、 最終承認を成立させない正しいFail-Closed状態です。

最終承認条件

検証項目 必須条件
Stage376結果 固定SHA256と一致
Stage372履歴 固定SHA256と一致し、変更されていない
RFC3161 対象、署名、証明書チェーンを検証済み
OpenTimestamps 公開アンカーと検証時刻を確認済み
共通対象 両方が同一Stage360 SHA256を参照
証明数 独立検証済み証明が2件
公開境界 生証明・秘密情報がdocs配下に存在しない

Stage377の判定

Decision 意味
timestamp_finalization_pending 両方ともまだ最終検証されていません。
rfc3161_verified_opentimestamps_pending RFC3161は検証済みですが、 OpenTimestampsが未確定です。
opentimestamps_verified_rfc3161_pending OpenTimestampsは検証済みですが、 RFC3161が未検証です。
dual_timestamp_final_acceptance_verified 両方が同一対象について独立に検証され、 新しい有効な最終承認が成立しました。
block ハッシュ、証明、公開境界などの 必須条件に違反があります。
